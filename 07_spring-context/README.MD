# Spring context

Spring, прежде всего, это про DI. Поэтому начнём разбираться с этой частью spring.

## Bean
Самая маленькая единица spring - это бин. 
Можно сказать, что это какой-то объект какого-то класса, но управляемый спрингом. 
Spring управляет их созданием и связями между ними.

## Configuration
Конфигурация spring - это, своего рода, чертёж, по которому spring соберёт нам программу.
Там мы можем описать бины и какой бин куда девать.
Есть 2 основных способа:
- через xml файл
- через аннотации в java коде

## Context
Контекст - главное место, где бины создаются, настраиваются и хранятся. 
Можно думать о нём, как об умном контейнере.

## Scopes
У бинов есть скоупы. Это, можно сказать, область жизни отдельного бина.
Проще описать на примерах:
- Singleton. Дефолтный и самый частоприменяемый скоуп. 
Означает, что в нашей программе создастся один экземпляр класса.
- Prototype. Означает, что при каждом запросе бина будет создаваться новый экземпляр класса.
В этом примере, например, есть Java конфиг с бином Product. 
Этот бин требуется для создания двух полок (Shelf). Для каждой полки создаётся отдельный экземпляр класса.

Немного спойлеров:
- Request scope. Означает, что на каждый http запрос будет создаваться отдельный экземпляр класса.
- Session scope. Отдельный экземпляр на отдельную пользовательскую сессию.
